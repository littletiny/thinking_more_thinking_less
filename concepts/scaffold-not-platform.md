# Scaffold, Not Platform / 脚手架，而非平台

## One Sentence
关键转变：不做通用平台（打不过 Dify/n8n/Coze），而是做普通人能用的独立脚手架——让每个人拥有自动化能力，而非提供平台服务。

---

## The Realization

### 之前的误区

```
试图构建：
- 一个通用的 DIY 开发平台
- 与 Dify/n8n/Coze 竞争
- 追求某种商业成功

结果：
- 打不过现有平台（生态、资源、易用性）
- 目标用户不清晰
- 陷入过度设计
```

### 现在的清醒

```
真正要做的：
- 不是平台，是脚手架
- 不是服务，是能力赋予
- 不是商业化，是普惠
- 不是给愿意折腾的人，是给真正的普通人

参考：OpenClaw
- 开源免费
- 普惠大众
- 降低门槛
- 每个人可以用
```

## OpenClaw 的启示

### OpenClaw 是什么

```
定位：AI Agent 框架（不是平台）
核心：让 AI 真正动手干活
特点：
- 开源免费（MIT 协议）
- 本地运行
- 普惠大众（GitHub 114,000+ stars）
- 技能系统（6000+ 插件）
- 屏幕操作（无需 API 集成）
成功关键：降低门槛，让普通人能用
```

### OpenClaw vs 平台

| 维度 | 平台（Dify/n8n/Coze） | 脚手架（OpenClaw） |
|------|---------------------|-------------------|
| **形态** | 中心化服务 | 分散式工具 |
| **控制** | 平台控制 | 用户控制 |
| **数据** | 在平台 | 在用户本地 |
| **门槛** | 需学习平台 | 开箱即用 |
| **目的** | 提供服务 | 赋予能力 |
| **商业模式** | SaaS 订阅 | 开源/免费 |

### 关键启示

```
OpenClaw 的成功不是因为技术最强
而是因为：
1. 普惠（人人可用）
2. 开源（无门槛获取）
3. 本地（数据属于自己）
4. 简单（降低使用门槛）
```

## Scaffold 的定义

### 不是平台，是什么？

```
平台：
- 你在我这里创建东西
- 我提供基础设施
- 我控制规则和收费
- 你依赖我

脚手架：
- 我给你工具
- 你在你的地方使用
- 我不管你怎么用
- 你不依赖我
```

### DIY 脚手架的形态

```
交付物：
├── 代码生成器（CLI/GUI）
├── 模板库（各种场景）
├── 部署脚本（一键运行）
├── 文档（普通人能懂）
└── Demo 展示（看见可能性）

不提供：
❌ 托管服务
❌ 账号系统
❌ 收费墙
❌ 平台规则

使用方式：
1. 用户下载/安装工具
2. 自然语言描述需求
3. 工具生成代码
4. 用户在本地运行
5. 完全属于用户
```

## The Shift

### 从平台思维到脚手架思维

| 平台思维 | 脚手架思维 |
|---------|-----------|
| 如何吸引用户到平台 | 如何让工具更容易被使用 |
| 如何收费 | 如何免费普惠 |
| 如何竞争 | 如何赋能 |
| 中心化 | 分散化 |
| 用户依赖平台 | 用户拥有工具 |
| 持续运营 | 一次性交付 |

### 核心问题转变

```
之前：
"如何做一个比 Dify 更好的平台？"

现在：
"如何做一个普通人能用的脚手架？"

关键差异：
- 不追求比平台好
- 追求让普通人能用
- 不追求商业成功
- 追求普惠价值
```

## Who Is It For?

```

❌ 程序员（他们直接用 OpenClaw/Cursor）
❌ 技术人员（他们用 n8n）
❌ 愿意折腾的人（他们自己找方案）

✅ 真正的普通人
```

```
- 自动记账
- 自动提醒缴费

- 自动提醒吃药
```

## Implementation Implications

### 关键改变

```
1. 不做 Web 平台
   - 不做账号系统
   - 不做托管服务
   
2. 做本地工具
   - 桌面应用（Electron/Tauri）
   - 或 CLI 工具
   - 用户下载即用

3. 开源免费
   - MIT 协议
   - 无收费墙
   - 社区驱动

4. 极致简单
   - 一个按钮生成
   - 自动处理部署
   - 普通人能懂

5. 完全属于用户
   - 生成后无依赖
   - 数据本地存储
   - 可自行修改
```

### 轻量级维护

```
只做：
- 核心代码生成器
- 模板库维护
- 文档和教程
- Demo 展示

不做：
- 托管服务
- 用户管理
- 收费系统
- 复杂运营

成本：
- 开发时间
- 模型 API 费用（可众筹/捐赠）
- 几乎零运营成本
```
