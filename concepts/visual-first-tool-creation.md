# Visual-First Tool Creation / 视觉优先的工具创建

## One Sentence
通过"随手拍 + 自然语言描述"作为输入，让非技术人员能够即时创建符合自己需求的记录类工具，实现从"观察到记录"的无缝闭环。

---

## Why It Matters

### 记录类产品的痛点

```
传统路径：
观察到想记录的东西 → 打开手机 → 找对应的 App → 发现没有合适的 → 
下载新 App → 注册 → 学习使用 → 终于开始记录 → 记录方式不符合需求 → 放弃

时间成本：5-15 分钟
成功率：< 20%
```

### 视觉优先的直觉性

```
人类认知顺序：
看到 → 想记录 → 描述看到的东西

而非：
想记录 → 找工具 → 学习工具 → 开始记录

视觉是最高效的信息输入方式，比文字描述更自然
```

## Source Context

From conversation: [mockcraft-design-discussion](../conversations/2026-03-09-mockcraft-design-discussion.md)

**Key case**: 讨论 MockCraft 个人/家庭场景时，提到"妈妈记录宝宝喂奶"场景，引发对"随手拍 + 自主开发"模式的思考。

**Key insight**: 跨领域的记录类产品（育儿、健身、装修、收藏、旅行）有共同的底层需求：结构化存储照片/数据 + 简单统计/回顾。视觉输入（拍照）+ 即时生成工具，可能比通用 App 更符合直觉。

```

├─────────────────────────────────────────────────────────────┤

│  系统自动识别：                                               │
│  • 图片内容（食物/数字/场景/物品）                            │

│                                                             │
└─────────────────────────────────────────────────────────────┘
                            ↓

│  Step 3: 工具生成（即时创建）                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  基于识别结果，自动生成记录工具：                               │
│                                                             │
│  数据模型：                                                  │
│  • 日期时间（自动）                                           │
│  • 食物名称（图片识别 + 用户确认）                            │
│  • 食量（图片估算 + 用户调整）                                │

│                                                             │
│  界面生成：                                                  │
│  • 快速记录入口（拍照即记录）                                  │

│  • 统计图表（本周吃了哪些食物）                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 即用即走（零学习成本）                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户无需：                                                  │
│  • 下载新 App                                                │
│  • 学习功能                                                  │
│  • 配置字段                                                  │
│                                                             │
│  立即可以：                                                  │
│  • 继续拍照记录                                              │
│  • 查看统计                                                  │
│  • 分享给家人                                                │
│                                                             │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 跨领域共性抽象

```
所有记录类产品的底层结构：

┌─────────────────────────────────────────┐
│  时间戳（自动）                          │
├─────────────────────────────────────────┤
│  主体（谁/什么）                         │
│  • 宝宝、我、物品、地点                   │
├─────────────────────────────────────────┤
│  动作/事件（做了什么）                    │
│  • 吃了、练了、去了、买了                 │
├─────────────────────────────────────────┤
│  度量（数值）                            │
│  • 体重、次数、金额、评分                 │
├─────────────────────────────────────────┤
│  媒体（证据）                            │
│  • 照片、视频、语音                       │
├─────────────────────────────────────────┤
│  标签/分类（便于检索）                    │
│  • 辅食类型、运动项目、地点类型            │
├─────────────────────────────────────────┤
│  备注（自由文本）                         │
└─────────────────────────────────────────┘

不同领域只是这些字段的组合和命名不同
```

## Technical Implementation

### 多模态 AI 技术栈

```
输入层：
├─ 图像输入（相机/相册）
├─ 语音输入（语音描述）
└─ 文本输入（补充信息）

理解层：
├─ 图像识别（物体检测、OCR、场景分类）
│   技术：GPT-4V / Claude 3 / Gemini
├─ 意图推测（根据场景判断记录类型）
│   技术：LLM + 场景模板库
└─ 信息提取（时间、地点、数值）
│   技术：EXIF 解析、GPS、NLP

生成层：
├─ Schema 生成（数据模型）
├─ UI 生成（界面布局）
└─ 工具组装（完整应用）
```

### 关键技术挑战

| 挑战 | 解决方案 |
|------|---------|
 图像识别准确率 | 多模态大模型 + 用户确认机制（AI 推测，人确认） |
 冷启动（无历史数据） | 领域模板库（育儿、健身等常见场景预设） |
 数据隐私（照片敏感） | 本地处理优先，云端仅用于模型推理（可选项） |
 多设备同步 | GitHub Gist / 自建同步服务 |

## Key Advantages

### 1. 零门槛

```
传统方式：
想到要记录 → 搜索 App → 下载 → 注册 → 配置 → 学习 → 开始记录
（15 分钟，80% 人放弃）

视觉优先方式：
看到 → 拍照 → 自动生成工具 → 立即记录
（30 秒，90% 完成）
```

### 2. 完全个性化

```
通用 App 的问题：
• 宝宝喂养 App：太多用不上的功能（睡眠、疫苗）
• 我只想记录辅食，但不得不看一堆广告

自主生成：
• 只生成你需要的字段
• 界面只显示你关心的统计
• 没有多余功能，没有广告
```

### 3. 随需而变

```
场景演化：
• 第 1 个月：只关心吃了什么
• 第 3 个月：加上体重、身高
• 第 6 个月：加上睡眠、大运动

工具演化：
随时说"加一个新字段"，30 秒后更新，数据保留

通用 App：
等开发商更新，或换另一个 App，历史数据丢失
```

## Potential Risks

### 1. 图像隐私

```
风险：用户随手拍，可能包含敏感信息（他人、地址、证件）

缓解：
• 默认本地处理，不上传原图
• 敏感内容检测（人脸、文字）提醒用户
• 提供"截图模式"（只取部分区域）
```

### 2. 识别错误

```
风险：AI 把"粥"识别成"汤"，导致数据错误

缓解：
• AI 推测 + 用户确认（默认高置信度才自动填充）
• 一键修改（不像传统 App 要进设置找）
• 学习用户习惯（第二次同样场景自动正确）
```

### 3. 工具碎片化

```
风险：每个场景生成一个独立工具，数据分散

缓解：
• 统一数据层（都存 GitHub Gist / 本地 IndexedDB）
• 跨工具关联（如"健身记录"关联"饮食记录"）
• 统一入口（一个 App 管理所有自建工具）
```

## Personal Notes

- 这可能是 MockCraft 最具颠覆性的方向：不是"做工具给普通人用"，而是"让普通人自己生成工具"
- 视觉输入比文字更自然，降低了"表达需求"的门槛
- 关键在于"即时"——看到 → 拍到 → 工具就有了，中间没有中断
- 隐私保护是成败关键，必须本地优先、用户掌控数据
- 这是一个"AI Native"的应用范式，没有 AI 之前不可能实现
