# Obsidian

## 一句话
一款基于本地 Markdown 文件的双向链接知识库工具，定位是"第二大脑"。

---

## 为什么重要

**核心需求**：数据主权。
- 知识应该比工具活得久
- 不依赖公司生存
- 随时可迁移

与 Evernote/Notion 的对比：本地优先、纯文本、完全可控。

## 讨论与挣扎

### 与 Git & Agent 的关系

**分层架构**：
```
Obsidian（前端）：人阅读、探索、可视化
    ↓ Markdown
Git（后端）：版本控制、安全、审计
    ↓ Agent 维护
Agent：自动维护、发现关联
```

**工作流**：
- 用 Agent 批量维护知识库（Git 保障安全）
- 用 Obsidian 阅读、探索
- 用 Git 记录版本

### AI 时代的地位

**潜在风险**：
- Agent 可以模拟 Obsidian 的核心功能（维护链接、生成索引）
- Agent 可以生成对话式知识库（问答而非只读）
- Obsidian 可能退化为"前端渲染器"

**防御优势**：
- 纯文本格式是 Agent 最容易处理的
- 本地优先与 Agent 本地运行兼容
- 数据主权在 AI 时代更重要

### 图谱的局限

- 笔记量大（>1000）时变成"毛球"
- 只反映链接关系，不反映语义相似度
- 需要人工维护 MOC，负担重

### 插件依赖的矛盾

- **核心价值**：纯文本，不锁定
- **但插件生态**：Dataview、Templater 等增加锁定

**策略**：核心内容保持纯 Markdown，插件只用于"视图"。

## 核心功能

### 双向链接
`[[概念名]]` - 促进知识关联，但链接是手动的。

### Graph View
可视化知识网络，但大数据量时性能下降。

### 本地优先
数据完全可控，即使 Obsidian 消失，文件还在。
