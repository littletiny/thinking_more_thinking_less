# Connection: Mock-Driven Validation × Natural Language Mock

## Relationship

**Natural Language Mock 是 Mock-Driven Validation 的技术基础。**

- **Mock-Driven Validation** (方法论): 用可触摸的 Mock 验证用户主观感受
- **Natural Language Mock** (技术实现): 用自然语言生成 Mock，将 Mock 成本降至趋近于零

---

## How They Enable Each Other

### Without NL-Mock

Mock-Driven Validation 面临成本瓶颈：
- 快速迭代需要频繁修改 Mock
- 传统 Mock 代码编写耗时
- 用户反馈循环被技术实现拖慢

### With NL-Mock

```
用户模糊描述需求
        ↓
AI 生成自然语言 Mock 描述
        ↓
生成交互原型（几行描述即可部署）
        ↓
用户体验并反馈
        ↓
AI 调整自然语言描述（秒级）
        ↓
新版本原型（秒级）
        ↓
...快速迭代直到对齐...
```

**关键转变**：Mock 修改从"代码工程"变成"文本编辑"

---

## Synergy Effects

| 维度 | 单独使用 | 结合使用 |
|------|----------|----------|
| **迭代速度** | 小时级 | 分钟级 |
| **用户参与度** | 需要等待技术实现 | 实时交互 |
| **探索范围** | 受限于 Mock 开发成本 | 可以探索多种可能性 |
| **失败成本** | 中等（需改代码） | 极低（改几句话） |

## Key Insight

这个组合的颠覆性在于：

> **需求验证和实现的边界变得模糊**

传统流程：
```
需求文档 → 设计稿 → Mock 开发 → 用户验证 → 修改需求 → 重新开发
   ↑__________________________________________________|
                    (长反馈循环)
```

NL-Mock 赋能的流程：
```
自然语言描述 → 即时原型 → 用户反馈 → 调整描述 → 新版本原型
      ↑_____________________________________________|
                      (短反馈循环)
```

**Mock 描述可以直接演化为真实实现的规格说明。**
